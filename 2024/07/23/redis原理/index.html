SDS

出现的由来

我们知道redis是使用c语言来编写的，但是redis并没有直接使用c语言中的字符串，因为c语言的字符串存在很多的问题：

  • 获取字符串的长度需要通过运算
  • 非二进制安全（以’\0’来结束标识）
  • 不可修改

数据结构源码

扩容规则

  • 如果新字符串小于1M，则新空间为扩展后字符串长度的2倍+1
  • 如果新字符串大于1M，则新空间为扩展后字符串的长度+1M+1 。称为内存预分配。

为什么要有扩容规则

程序分配空间需要从用户态切换到内核态，调用相关提供的api来帮助分配空间，这一操作比较费时，所以我们应该尽可能的避免变态操作。

优势

  1. 获取字符串长度的时间复杂度为o1
  2. 支持动态扩容
  3. 减少内存分配的次数
  4. 二进制安全

IntSet

IntSet是redis中set集合的一种实现方式，基于整数数组来实现的，并且长度可变，有序等特征。

数据结构源码

intset中数据的大小是由encoding来决定的，决定是存16位还是32位还是64位，而contents只是一个指针，用来指向第一个元素的地址

为了方便查找，redis会将intset中所有的整数按照升序的次序依次保存在维护的数组中

插入

升级

当插入的数字超出了encoding指定的整数类型大小，intset会自动升级编码方式到合适大小，流程如下：

  1. 升级编码类型，并按照新的编码方式及元素个数扩容数组
  2. 倒序依次将数组中的元素拷贝到扩容后的正确位置
  3. 将待添加的元素放入数组末尾/队首（因为这个元素已经是最大/最小的元素了）
  4. 最后将intset的encoding属性改为对应的编码标识，将length属性+1

特点

  1. redis会确保intset中元素的唯一、有序
  2. 具备类型升级机制，可以节省内存空间
  3. 底层采用二分查找来查询

Dick

redis是一个键值类型的数据库，我们可以根据键实现快速的增删查改。而键与值的映射关系是通过Dict来实现的。

Dict由三部分组成：哈希表、哈希节点、字典

void * 代表任意类型的指针 union代表value的类型可以是四个中的任意

哈希运算

当我们向Dict中添加键值对时，redis首先根据key计算出hash值（h），然后利用h&sizemask来计算元素应该存储到数组中的哪一个索引位置。

扩容

每次新增元素时检查负载因子（loadfactor = used/size）满足下列俩种情况会触发扩容：

  1. loadfactor >= 1 并且服务器没有执行 BGSAVE 或 BGREWRITEAOF 等后台进程。
  2. loadfactor > 5

可以进行二分查找的有序链表。

数据结构示意图

由图可知，对应的数据结构为：

查找

1、从最左上节点开始查找

2、 需要查找的值 大于 当前值：继续比对这一层链表的下一个元素 然后循环

3、需要查找的值等于当前元素 ，直接返回

4、需要查找的值小于当前元素，回到这一层链表的上一个元素并下降一层继续比对

我要查找数据F，我的出发点在最左上的节点A

第一步：比对A和F，发现A比F小，比对下一个元素G

发现F小于当前值G，回到元素A，并降层，当前元素的下一个节点E 和 查找元素F 进行比较 结果是 比对下一个元素G，结果依然是降层，一直降到第0层，找到F和F进行比较，返回结果。
